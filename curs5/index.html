Noțiuni importante

Paritatea unui număr

Pentru a verifica că un număr este par în C/C++ vom folosi operatorul % (restul împărțirii).

Știm de la matematică că un număr par se împarte exact la 2. În alte cuvinte, un număr par este un număr care, la împărțirea cu 2, are restul 0 (altfel nu s-ar fi împărțit exact).

Aplicând ceea ce am observat în cod, expresia pentru verificarea parității unui număr natural sau întreg este:

x % 2 == 0

Observație! Pentru numerele reale nu se pune problema verificării parității pentru că operatorul % NU poate fi aplicat operanzilor reali. Prin urmare dacă avem de-a face cu un număr real, acesta trebuie convertit la un tip de date convenabil (int, unsigned, etc.) pentru a face verificarea.

Accesarea ultimei cifre a unui număr natural sau întreg

În continuare, prezentăm altă aplicație a operatorului modulo.

Ca să accesăm ultima cifră a unui număr în C/C++ vom folosi expresia:

x % 10

Hai să luăm câteva exemple:

  • \(123 \% 10 = 3\)
  • \(987 \%10 = 7\)
  • \(6 \% 10 = 6\)

Observăm că restul împărțirii prin \(10\) a oricărui număr ne dă chiar ultima cifra a acestuia.

Ștergerea ultimei cifre a unui număr natural sau întreg

Ultima noțiune pe care o prezentăm este cum se șterge ultima cifra a unui număr (care mai poate fi numită ștergerea cifrei unităților sau ștergerea celei mai nesemnificative cifre a unui număr).

Expresia pentru aceasta este:

x / 10

Adică partea întreagă a împărțirii numărului la \(10\). Cum functionează asta? Prima dată efectuăm, ca la mate, împărțirea la 10. Apoi, când aplicăm partea întreagă, pe noi ne interesează partea numărului de dinaintea virgulei. Dacă numărul se împarte exact la \(10\) partea întreagă nu schimbă nimic. Hai să vedem niște exemple

  • \([\frac{123}{10}] = [12,3] = 12\)
  • \([\frac{987}{10}] = [98,7] = 98\)
  • \([\frac{100}{10}] = [10] = 10\)

Partea II:

Se consideră următoarele întrebări cu răspuns multiplu. Răspunsul este unic și există pentru toate întrebările, iar acesta consistă prin alegerea literei corespunzătoare afirmației corecte (A, B, C sau D).

1. Se consideră următoarea expresie C/C++, unde variabilele \(x\) și \(y\) sunt declarate în prealabil și sunt numerele reale:

(x + 5) / y + 7

Care este expresia matematică echivalentă cu cea prezentată mai sus?

  • A). \(\dfrac{x + 5}{y + 7}\)
  • B). \(\dfrac{x + 5}{y} + 7\)
  • C). \(x + \dfrac{5}{y + 7}\)
  • D). \(x + \dfrac{5}{y} + 7\)

2. Considerăm următoarea înșiruire de instrucțiuni:

int n, a, b, c;
cin >> n;
a = b = c = 0; // Initializare cu 0

// Operatia 1
a += n * (n + 1) / 2;

// Operatia 2
int i = 1;
while (i <= n) {
    b += i;
    i++;
}

// Operatia 3
c += 55

cout << a << ' ' << b << ' ' << c << '\n';

Care dintre următoarele afirmații este FALSĂ?

  • A). Secvența de instrucțiuni calculează în variabilele întregi \(a\) și \(b\) suma primelor \(n\) numere consecutive naturale;
  • B). Dacă pentru variabila \(n\) se citește valoarea \(10\), atunci, la finalul secvenței de instrucțiuni, se vor afișa trei valori identice, separate printr-un spațiu;
  • C). Dacă pentru variabila \(n\) se citește valoarea \(9\), atunci, la finalul secvenței de instrucțiuni, se vor afișa numerele 45 45 55;
  • D). Se vor produce exact două erori: una de sintaxă pentru că nu există operatorul += și una la momentul compilării codului sursă.

3. Vacile fermierului Ion au găsit pe pajiște un număr de exact \(5\) cifre. În mitologia bovină se spune că dacă reușești să extragi cifra sutelor din număr, atunci vei avea parte de prosperitate și multe căpițe de fân. Dacă numărul găsit îl notăm \(n\) și știm că este natural, atunci ce expresie C/C++ trebuie să folosească văcuțele ca să extragă cifra sutelor?

  • A). n / 10 % 10;
  • B). n / 100 % 10;
  • C). n / 10000;
  • D). n % 10 / 100.

4. Vaca Besi a găsit depozitul unde fermierul Ion ascunde L'Herbe (iarbă deluxe). Problema, însă, este că ușa este închisă de un lacăt care se deschide dacă și numai dacă poți răspunde instantaneu la o întrebare de tipul:

Este numărul \(x\) divizibil cu \(3\) sau numărul \((x + 1)\) divizibil cu \(5\)?

Besi a scris tot programul C++, înafară de expresia indicată prin ...:

#include <iostream>

using namespace std;

int main() {
    int x;
    cin >> x;
    if (...) // Expresia
        cout << "CORECT";
    else
        cout << "GRESIT :(";
    cout << '\n';
    return 0;
}

Care dintre următoarele expresii rezolvă corect întrebarea pusă de lacăt?

  • A). x % 3 == 0 && (x + 1) % 5 == 0;
  • B). x % 3 == 0 || x + 1 % 5 == 0;
  • C). x % 3 == 0 || (x + 1) % 5 == 0;
  • D). x % 3 == 0 && x + 1 % 5 == 0.
